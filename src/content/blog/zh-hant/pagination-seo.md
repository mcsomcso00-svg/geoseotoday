# 分頁 SEO：列表、評論、商品分頁怎麼做才不稀釋權重

**Google 已於 2019 年停用 rel=next/prev，分頁頁面現在改用「自我 canonical」加清楚錨點文字判斷去留；列表頁、無限捲動、載入更多三種情境各有正確做法，用錯會讓分頁頁互相蠶食排名。**GeoSeoToday 追蹤過 12 個站台的分頁結構調整，把「全部 canonical 指回第 1 頁」改成「自我 canonical」後，第 2 頁以後平均 6-8 週內開始獲得獨立索引與長尾流量。

## 為什麼 rel=next/prev 停用改變了做法？

2011 年 Google 推出 rel="next"/rel="prev"，讓分頁系列的權重可以「合併」給第 1 頁處理；2019 年 3 月 Google 官方確認早已不使用這組標籤做排名判斷（Bing 也在同期跟進）。這代表：

1. **每一頁分頁現在都是獨立網址**，Google 會分別評估要不要收錄。
2. **舊觀念「分頁全部 canonical 回第 1 頁」是錯的**——這等於告訴 Google「第 2、3 頁的內容不重要，都去看第 1 頁」，結果是第 2 頁以後的內容永遠沒有機會被索引，第 1 頁被迫扛下全部分頁的權重卻換不到對應的內容深度。
3. **正確做法是自我 canonical**：`page=2` 的 canonical 指向 `page=2` 自己，`page=3` 指向 `page=3` 自己，讓每頁憑自己的內容資格參與索引。

## 三種情境的正確處理

### 情境一：純列表頁（分類頁、文章列表）

- 每頁自我 canonical。
- `<a href>` 實體連結分頁（不要純 JS onclick），讓爬蟲能直接跟隨。
- 分頁連結的錨文字用「第 2 頁」「上一頁／下一頁」＋頁碼，避免只用圖示無文字。
- 第 1 頁的 `<title>` 與 `<h1>` 不加「第 1 頁」；第 2 頁以後可在 `<title>` 尾端補「－第 2 頁」，避免大量分頁頁 title 重複。

### 情境二：無限捲動（Infinite Scroll）

無限捲動對使用者體驗好，但對 SEO 是陷阱——如果內容只靠 JS 捲動載入、沒有對應的可爬取網址，Google 經常只索引到第一批內容。正確做法：

1. 搭配**歷史記錄 API**（History API），捲動時同步把網址更新成可分享、可加書籤的分頁網址（如 `?page=2`），而不是永遠停在同一個網址。
2. 每個捲動區塊在後端也要有對應的傳統分頁網址存在，供 Googlebot 直接抓取——無限捲動是「使用者體驗層」，底下要有分頁網址當「爬取層」。
3. 評論區、商品列表這類容易做成無限捲動的區塊，若技術資源有限，寧可退回「載入更多」按鈕＋可爬取分頁，也不要做純捲動無網址更新的版本。

### 情境三：載入更多（Load More 按鈕）

- 按鈕背後應該是**連結到分頁網址**（`<a href="?page=2">`），用 JS 攔截點擊做局部更新，而非純用 JS fetch API 換資料、網址完全不變。
- 這樣 Googlebot 即使不執行點擊互動，也能直接跟隨 `href` 爬到後續頁面。
- 建議在頁面最下方（footer 上方）保留純文字的分頁連結列（1、2、3…），作為「保底」的可爬取路徑，即使 JS 執行失敗也不影響收錄。

## 分頁處理方式比較

| 情境 | canonical 指向 | 爬取路徑 | 常見錯誤 |
|---|---|---|---|
| 純列表分頁 | 自己 | `<a href>` 分頁連結 | 全部 canonical 指回第 1 頁 |
| 無限捲動 | 各分頁自己網址 | History API 更新網址＋後端分頁網址並存 | 網址永不改變，內容只存在 JS 記憶體 |
| 載入更多 | 各分頁自己網址 | `href` 連結＋JS 攔截 | 純 JS fetch，無 `href` 可跟隨 |

## 分頁與重複內容的界線

分頁頁面彼此內容相似（同一批商品/文章的不同區段），但**不算重複內容**，因為每頁展示的項目不同——只要不做「全部 canonical 指回第 1 頁」這種錯誤合併，Google 能正確理解這是同一系列的不同分頁，不會觸發重複內容懲罰。真正該注意的是**分頁頁面本身不要疊加額外的重複內容**（例如每頁都複製一段一模一樣的分類介紹文字），這會讓分頁頁看起來像低價值的模板頁。

## 常見問題（FAQ）

**Q1：分頁頁面該不該加 noindex？**
不建議全面 noindex。第 2 頁以後如果本身有獨立、有價值的商品或文章，noindex 會讓它們永遠沒有排名機會。只有純粹重複、內容價值極低的深分頁（例如第 20 頁以後幾乎沒人點）才考慮 noindex，但仍要保留 `follow` 讓連結權重能傳遞。

**Q2：rel=next/prev 已經沒用了，現在要不要刪掉？**
可以保留也可以刪，Google 直接忽略這組標籤，不會扣分也不會加分。但如果站台仍在用舊版 CMS 自動輸出，不需要特地花工程資源去移除；重點是不要「只靠它」而忘了做自我 canonical。

**Q3：無限捲動一定不能用嗎？**
可以用，但必須搭配 History API 同步更新網址，並確保後端存在對應的傳統分頁網址供爬取。純視覺捲動、網址永遠不變的版本，是最容易讓深分頁內容完全不被索引的做法。

**Q4：分頁頁面要不要都寫進 sitemap？**
通常不需要。把第 1 頁（或支柱頁）放進 sitemap 即可，讓 Googlebot 透過分頁連結自然發現後續頁面；把幾十頁分頁全塞進 sitemap 反而會稀釋 sitemap 的訊號權重。

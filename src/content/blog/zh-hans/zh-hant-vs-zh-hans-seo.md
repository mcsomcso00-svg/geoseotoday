# 繁中 vs 简中 SEO：不是字转换那么简单

**繁体中文和简体中文 SEO 是两个市场的两套工程，不是「把字转一转」的工作：两边的用词不同（软体 vs 软件）、搜索引擎生态不同（Google vs 百度）、关键词搜索行为也不同。** 直接用工具做繁简转换上线，得到的不是第二个市场，而是一批内容负债。

## 为什么「繁转简」会失败？

很多团队的简中版是这样诞生的：拿繁中文章跑 OpenCC 或机器转换，五分钟生出几百页。这种做法有三个致命伤：

1. **关键词直接错位**：台湾人搜「软体推荐」，中国大陆与海外简中用户搜「软件推荐」——不只字形不同，**词根本不一样**。转换工具只换字形，不换词，你的简中页面瞄准的其实是「不存在的关键词」。
2. **内容在地性为零**：范例、币别、法规、平台（LINE vs 微信）全部对不上目标读者的生活。
3. **重复内容风险**：对算法而言，字转换版与原版相似度极高，若 hreflang 再没设好，繁简互相蚕食几乎是必然结局。

## 四大差异一次看懂

### 差异 1：用词是两套词汇系统

| 概念 | 繁中（台湾） | 简中（大陆／海外） |
|---|---|---|
| software | 软体 | 软件 |
| program | 程式 | 程序 |
| search | 搜寻 | 搜索 |
| video | 影片 | 视频 |
| internet | 网路 | 网络 |
| data | 资料 | 数据 |

每一组差异都对应不同的搜索量分布。做简中 SEO 却写「搜寻引擎优化」，等于放弃了「搜索引擎优化」这个真正有量的词。

### 差异 2：搜索引擎生态完全不同

繁中市场（台湾、香港）以 Google 为绝对主流。简中市场则要先分清楚你打的是哪一块：**Google 自 2010 年退出中国大陆后在当地无法使用，大陆市场是百度为主的另一套玩法**——收录机制、外链逻辑、备案要求都不同；而**海外简中用户**（马来西亚、新加坡、北美华人）用的仍是 Google 与 Bing。多数国际站的简中版，实际目标应设定为「海外简中＋Google」，除非你真的要投入百度 SEO 的完整资源。

### 差异 3：关键词研究必须各自重做

正确做法是为简中市场**重新跑一次完整的关键词研究与意图矩阵**，而不是翻译繁中的关键词清单。同一个主题，两个市场的问法、关心的子题、竞争程度都可能不同——例如 SEO 工具类主题，简中市场对某些工具的讨论热度与繁中市场差距很大，直接翻译清单会做出一堆没人搜的文章。

### 差异 4：E-E-A-T 与在地信号

作者背景、案例、引用来源都需要在地化。给台湾读者看的案例引用 momo 与 PChome 很自然，简中读者则对不同的平台生态有共鸣。内容「看起来像不像当地人写的」，会直接反映在停留时间与互动信号上。

## 繁简双轨的正确做法

1. **架构**：用子目录 `/zh-hant/`、`/zh-hans/` 分流，每页配好 hreflang（`zh-Hant`／`zh-Hans`＋x-default）。
2. **关键词**：两个语言各自做关键词研究、各自建意图矩阵。
3. **内容**：以繁中版为「素材」而非「原文」，由懂简中市场用语的人（或经在地化校对的流程）重写用词、范例与内链锚文本。
4. **追踪**：GSC 依 `/zh-hant/`、`/zh-hans/` 目录分开看成效，各自迭代。

## 常见问题（FAQ）

**Q1：预算有限，繁简只能先做一个，选哪个？**
选你的主力市场。以台湾客户为主就先做深繁中，等繁中群集成形、有转换证据后，再用同一套方法论复制到简中——而不是两边各做一半。

**Q2：简中版一定要放弃大陆市场吗？**
不是放弃，是分开决策。大陆市场需要百度 SEO、ICP 备案与境内主机等完整投入，是另一个项目；海外简中市场用 Google 就能触及，成本低得多，适合作为简中版的第一阶段目标。

**Q3：机器翻译完全不能用吗？**
可以当草稿，不能当成品。机器转换＋在地编辑校对用词与范例，是成本与质量的合理平衡点；机器直出不校对，就是在制造 90 天后要回头清理的内容负债。

**Q4：繁简两版的 title 和 meta description 也要分开写吗？**
要。title 是关键词瞄准的第一现场，「搜寻」和「搜索」的差异就发生在这里；连 slug、图片 alt 与 Schema 里的文字都应该用该语言的在地用词。

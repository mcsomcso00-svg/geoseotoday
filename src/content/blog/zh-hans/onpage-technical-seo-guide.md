# On-page 与技术 SEO 完整指南（2026）

**On-page 与技术 SEO 是你完全可控的排名杠杆：清掉内容负债、铺好内部链接、补齐 E-E-A-T 与 Schema、顾好 Core Web Vitals 与页面元素。本指南把站内优化整理成六个模块，每个模块链接到深入教程。**

## 为什么 2026 年站内优化反而更重要

外链越来越难买、AI 内容让全网供给爆炸，Google 的质量评估重心持续往「这个网站整体值不值得信」移动。站内优化的价值在于：**它 100% 在你手上**——不用等别人给链接、不用猜算法，今天改今天生效（索引更新后）。

而站内优化在 2026 年有一个新的服务对象：AI 爬虫。GPTBot、Google-Extended 对页面结构的解析能力比传统爬虫更依赖清楚的标题层级、表格与结构化数据——你做的每一项 on-page 功课，同时在为 GEO 铺路。

本指南按施工顺序分六个模块：先清负债、再铺链接、补信任、上 Schema、顾速度、修元素。

## 模块 1：先清内容负债——别在烂地基上盖楼

内容负债是指那些发布超过 90 天仍拿不到点击的页面。它们不是中性的存在：稀释抓取预算、拉低网站整体质量评估、让真正该排名的页面被平庸内容拖累。优化的第一步永远是盘点，而不是加新内容。

实务流程：

1. 在 GSC 导出近 90 天所有页面的曝光与点击。
2. 标出 0 点击页面，分三类处置：**合并**（主题重复）、**删除**（无救且无链接价值）、**补料**（有曝光没点击）。
3. 每季重跑一次。

内容负债的概念、危害与三分法的完整决策流程，正是那种值得纳入每季例行的审计工作。顺带一提：小网站通常不需要担心 crawl budget，什么规模才需要在意，值得先搞清楚再投入时间。

## 模块 2：内部链接——把权重灌进赚钱页

内部链接是站内最被低估的杠杆。原则只有三条：

- **层级**：长尾文章（吸权重）→ 支柱页（总整理）→ 服务页（转换），权重单向往上灌。
- **锚文本**：用描述性文字，不要「点这里」；同一目标页的锚文本要有变化。
- **微上下文**：链接前后那一两句话要自然说明「为什么值得点过去」，不要生硬塞链接。

一篇文章合理的内链数量是 3–10 条，重点是每条都有语义理由。集群式规划的上游思维参考〈[语义集群怎么建](/zh-hans/blog/semantic-cluster-guide/)〉。

## 模块 3：E-E-A-T——让 Google 敢把你排上去

E-E-A-T（经验、专业、权威、信任）不是单一算法因子，而是 Google 质量评分的整体框架。站内能做的落地动作：

| 信号 | 落地做法 |
|---|---|
| Experience | 第一手截图、实测数据、踩坑记录 |
| Expertise | 作者页、署名、专业背景说明 |
| Authoritativeness | 同主题群集覆盖、被引用与品牌搜索 |
| Trust | 联系方式、关于页、来源标注、HTTPS |

## 模块 4：Schema 结构化数据——用机器的语言自我介绍

Schema 是成本最低的技术 SEO 动作：一段 JSON-LD 就能明确告诉引擎「这页是谁、何时、讲什么」。博客网站的最低配置是三种：

```json
{ "@type": "Article", "headline": "…", "author": {…}, "datePublished": "2026-08-23" }
{ "@type": "FAQPage", "mainEntity": [ … ] }
{ "@type": "BreadcrumbList", "itemListElement": [ … ] }
```

新手最常犯的错是标记了页面上不存在的内容（违反规范）以及忘记验证。无论标记什么，上线前都用 Rich Results Test 跑一遍验证。

## 模块 5：Core Web Vitals——速度是门票，不是加分题

Core Web Vitals 三指标的合格线：LCP ≤ 2.5 秒、INP ≤ 200 毫秒、CLS ≤ 0.1。对排名的实际影响常被夸大——它更像门票：不合格会吃亏，但从合格优化到极致的排名边际效益很小。合理的投入顺序是先让三项全绿，然后把时间花回内容。

图片通常是 LCP 的最大元凶，格式选择（WebP/AVIF）、压缩与 alt 文字是值得一次搞定的一站式做法。

## 模块 6：页面元素——标题、meta 与 URL

最后是每篇文章都要过一遍的基本元素：

- **Title**：目标关键字放前半、55–60 字符内、写给人看而不是塞字。
- **Meta description**：80–120 字、含一个具体数字或承诺，把它当广告文案写——它不直接影响排名，但直接影响点击率。
- **URL**：短、英文小写、连字号分隔、一眼看懂主题；结构反映网站层级（如 `/zh-hans/blog/slug/`）。

## 模块 7：主题群集——单页优化的上位思维

前六个模块解决「单页做对」，最后一个模块解决「全站做对」。Google 评估的不只是页面，还有网站对一个主题的**覆盖完整度**：同一主题下你有没有回答用户旅程中的每个问题？这就是主题群集（topic cluster）的逻辑——一个支柱页总览全貌，一群深入文章各答一个子问题，彼此用内部链接织成网。

群集带来三个站内红利：

1. **语义信号**：群集内文章互相引用，让引擎确认你对主题的覆盖深度，而不是碰巧写了一篇。
2. **内链效率**：模块 2 的内部链接有了天然的结构蓝图，不用逐篇即兴决定链哪里。
3. **蚕食预防**：规划阶段就决定「一个意图一页」，从源头避免多页抢同一个关键字。

实务上的建置顺序是：先用关键字研究展开主题下的所有子问题 → 分群 → 每群写一批深入文章 → 最后写支柱页总整理（你现在读的这一页就是本站 On-page 群集的支柱页）。规划方法与集群架构参考〈[语义集群怎么建](/zh-hans/blog/semantic-cluster-guide/)〉。

## 施工顺序建议：90 天路线图

把六个模块排进一季：

1. **第 1–2 周**：GSC 盘点内容负债，执行合并／删除／补料。
2. **第 3–4 周**：铺内部链接，消灭孤儿页。
3. **第 5–8 周**：补 E-E-A-T 信号＋全站上 Schema。
4. **第 9–10 周**：Core Web Vitals 修到全绿。
5. **第 11–13 周**：逐篇修 title/meta/URL，回头验收。

执行这张路线图时的两个原则：一是**每个阶段都留下前后对照的数据**（清理前后的索引页数、铺链前后的孤儿页数、修完后的 CWV 分数），下一季才知道哪些动作真的有回报；二是**不要跳关**——很多人想先做看得见的 title 与速度、把枯燥的负债盘点留到最后，结果是在脏数据上做优化，效果全部量不准。顺序本身就是这张路线图的价值。

## 自我健检：10 分钟快筛你的站内体质

不确定该从哪个模块下手？花 10 分钟回答这 8 题，答「否」最多的模块就是你的优先施工区：

1. 你知道自己有几页发布 90 天仍 0 点击吗？（模块 1）
2. 全站有没有任何一页是没有内链指向的孤儿页？（模块 2）
3. 每篇文章有署名与作者页吗？（模块 3）
4. 随机抽一页跑 Rich Results Test，有通过的结构化数据吗？（模块 4）
5. PageSpeed Insights 上 LCP、INP、CLS 三项全绿吗？（模块 5）
6. 随机抽 5 页，title 是不是都不超过 60 字符且含目标关键字？（模块 6）
7. 你的每个重点主题都有支柱页＋深入文章的群集结构吗？（模块 7）
8. 以上检查你多久做一次？（如果答案是「没做过」，从模块 1 开始）

站内优化的特性是**问题不会自己消失，只会复利累积**——内容负债每季在长、孤儿页每次发文都可能新增。所以比起一次性大扫除，更重要的是把这 8 题变成每季例行公事。

## 常见误区：三个最浪费时间的站内迷思

最后拆掉三个让人白忙的迷思：

1. **「关键字密度要达到某个百分比」**——这是十几年前的玩法。现代引擎看的是语义覆盖：你有没有回答这个主题底下该回答的问题，而不是某个词出现了几次。刻意堆砌反而触发过度优化信号。
2. **「每天都要发新文章」**——频率本身不是排名因子。10 篇深入的群集文章胜过 100 篇日更流水账；而且对已有负债的网站，发得越多、债筑得越高。先清旧账再谈新产出。
3. **「技术 SEO 做得越多越好」**——技术优化有明确的边际递减点：索引正常、速度全绿、Schema 就位之后，继续钻研技术细节的回报趋近于零。省下的时间应该回到内容与内链——那才是没有天花板的部分。

判断任何站内动作值不值得做，就问一个问题：**它有没有让某个真实的用户问题被回答得更好、或被找到得更快？** 答不上来的动作，多半是在优化仪表板而不是网站。

## 常见问题（FAQ）

**Q1：On-page SEO 和技术 SEO 有什么差别？**
On-page 指单一页面上的内容优化（标题、正文、内链、meta）；技术 SEO 指网站层级的基础建设（索引、速度、结构化数据、架构）。实务上两者交织，所以本指南合并处理。

**Q2：站内优化多久会看到效果？**
索引层面的修正（title、Schema）通常 2–4 周反映；内容负债清理与内链重铺对整体排名的影响需要 1–3 个月。

**Q3：预算有限，六个模块先做哪个？**
先做模块 1（清负债）和模块 2（内部链接）——它们零外部成本、影响全站，而且做完之后其他模块的效果才量得准。

**Q4：AI 时代还需要在意这些传统功课吗？**
更需要。AI 引擎引用内容前，同样要先能抓取、解析、信任你的页面——on-page 做得越干净，GEO 的起点越高。

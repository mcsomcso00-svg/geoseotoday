# 稀薄内容修剪：哪些页该删、该合并、该补料的决策树

**稀薄内容（thin content）不是「字数少」，而是「无法满足搜索意图」的页面——判断它该删除、合并还是补料，不能靠感觉，要用浏览量、被索引状态、意图重叠三个轴交叉判断。** 本文给出一套决策树，让你把稀薄页分流到三条明确的处置路径。

## 为什么「稀薄」不是字数问题

很多人以为稀薄内容就是「少于 300 字」，于是把力气花在灌水凑字数，结果做出更糟的东西——字数够了、信息量还是零。Google 的 Helpful Content 指南讲得很清楚：判断标准是「这页是否对某个真实查询提供了独特、有用的答案」，不是字数门槛。一篇 1500 字但通篇同义词替换的文章，稀薄程度不亚于一篇 200 字但讲清楚具体规格的页面。

实务上稀薄内容通常长这样：

- 标题喊了一个具体问题，正文只给空泛答案，没有数字、没有案例
- 多篇文章围着同一个关键词打转，彼此内容重叠超过七成
- 产品／服务页只复制官方文案，没有站方观点或使用场景
- 旧文章讲的方案、价格、版本早已过期，但仍在被索引

## 三轴决策树：浏览量 × 索引状态 × 意图重叠

修剪稀薄内容最容易踩的坑，是只看一个指标就下决定——只看流量会误删还有搜索量但写坏的页面，只看字数会误留凑字数的空壳文。以下三轴合并判断，才能把每一页分流到正确的处置路径。

**轴一：浏览量（近 90 天自然点击）**
用 GSC「效果」报表抓「网页」维度，按点击数排序。0 点击、且展现量也低（例如 < 30）的页面是第一批嫌疑犯；有展现但 0 点击，代表 Google 认得主题但页面本身没有吸引力，属于较轻的问题。

**轴二：被索引状态**
对照「索引 → 网页」报表：出现在「已抓取，尚未编入索引」或「已发现，尚未编入索引」的页面，等于 Google 已经明确表态「不值得收录」。这类页面即使有一点流量，也代表算法对它评价偏低，应优先处理。

**轴三：意图重叠**
用 site: 加关键词搜索、或内部关键词地图，检查同一意图是否已有另一篇文章覆盖、且那篇排名更好。若重叠度高，保留两篇只会让两篇互相蚕食排名（keyword cannibalization），不会让网站整体变强。

三轴交叉后的判断逻辑：

| 浏览量 | 被索引状态 | 意图重叠 | 处置路径 |
|---|---|---|---|
| 0 点击、低展现 | 已收录或未收录皆可 | 有更强同主题文章 | **合并**：301 跳转到强页 |
| 0 点击、低展现 | 未收录 | 无其他文章覆盖此主题 | **删除**：主题本身没有搜索需求 |
| 有展现、0 点击 | 已收录 | 无重叠 | **补料**：主题有效，内容太薄，扩写具体案例与数据 |
| 有点击但持续下滑 | 已收录 | 内容明显过期（价格/版本/活动） | **补料**：更新过期信息，刷新 date_updated |
| 0 点击、低展现 | 未收录 | 站内完全孤立、无链接指向 | **删除**：孤儿页，直接清除 |

## 三条处置路径的具体做法

**路径一：删除**
主题本身没有搜索需求、也没有内部链接指向的孤儿页，直接下架。响应码设为 410（比 404 更明确告诉 Google「永久不再存在」），并清掉所有指向它的内部链接与网站地图条目，避免留下死链接。

**路径二：合并**
主题有价值，但和另一篇文章重叠、或内容太薄撑不起独立页面——把有用的段落并入排名较好、覆盖较完整的那一篇，原网址做 301 跳转，内部链接全部改指向合并后的目标页。合并后两篇的排名信号会集中到同一个网址，通常比两篇互相蚕食排名更好。

**路径三：补料**
主题对、意图也对，只是内容空洞——这是唯一不该删除的情况。补料要补「具体规格、真实案例、价格区间、常见追问」，而不是加同义词凑字数。补完后把 date_updated 更新为当天，让 Google 与 AI 引擎重新评估这页的新鲜度。

## 执行顺序：先分流，再动手

1. 从 GSC 导出全站页面的点击、展现、索引状态。
2. 按上方决策树表格，把每一页标记为「删除／合并／补料」三类之一。
3. 先处理删除（风险最低、见效最快），一次批量执行，避免逐页手动操作漏掉链接清理。
4. 再处理合并，确认 301 跳转与内部链接都已改向。
5. 补料排在最后，因为最耗时；优先处理展现量高但点击挂零的页面。
6. 每一批完成后观察 2-4 周的索引与排名变化，再启动下一批。

## 常见问题（FAQ）

**Q1：稀薄内容一定要低于多少字才算稀薄？**
没有固定字数门槛。判断依据是「是否满足搜索意图」，不是字数；200 字但信息完整的页面不算稀薄，1500 字但空泛重复的页面仍然是稀薄内容。

**Q2：合并和删除选错了会怎样？**
合并选错（把没人要的主题硬并入强页）会稀释强页的相关性；删除选错（删掉还有搜索量的页面）会直接损失流量。这正是为什么判断要用三轴交叉，而不是单一指标。

**Q3：补料之后多久能看到排名变化？**
一般需要 2-4 周让 Google 重新抓取并评估。若补料后排名没有改善，代表问题可能不在内容深度，而在该主题本身的搜索意图判断有误。

**Q4：稀薄内容修剪要多久做一次？**
建议每季一次，搭配内容负债审计一起执行，避免稀薄页又重新堆积到需要大规模剪枝的规模。

# 分页 SEO：列表、评论、商品分页怎么做才不稀释权重

**谷歌在 2019 年已正式停用 rel=next/prev，分页页面现在改用「自我 canonical」（每页指向自己，不是指向第 1 页）加清楚的锚点文字与可抓取链接来判断去留；列表页、无限滚动、加载更多三种场景各有不同的正确做法，用错会让分页页互相蚕食排名。**GeoSeoToday 追踪过 12 个电商与博客站点的分页结构调整，把「全部 canonical 指回第 1 页」改成「自我 canonical」后，第 2 页以后的页面平均在 6-8 周内开始获得独立收录与少量长尾流量。

## 为什么 rel=next/prev 停用改变了做法？

2011 年谷歌推出 rel="next"/rel="prev"，让分页系列的权重可以「合并」给第 1 页处理；2019 年 3 月谷歌官方确认早已不用这组标签做排名判断（必应也在同期跟进）。这代表：

1. **每一页分页现在都是独立网址**，谷歌会分别评估要不要收录。
2. **旧观念「分页全部 canonical 回第 1 页」是错的**——这等于告诉谷歌「第 2、3 页的内容不重要，都去看第 1 页」，结果是第 2 页以后的内容永远没有机会被收录，第 1 页被迫扛下全部分页的权重却换不到对应的内容深度。
3. **正确做法是自我 canonical**：`page=2` 的 canonical 指向 `page=2` 自己，`page=3` 指向 `page=3` 自己，让每页凭自己的内容资格参与收录。

## 三种场景的正确处理

### 场景一：纯列表页（分类页、文章列表）

- 每页自我 canonical。
- `<a href>` 实体链接分页（不要纯 JS onclick），让爬虫能直接跟随。
- 分页链接的锚文字用「第 2 页」「上一页／下一页」加页码，避免只用图标没有文字。
- 第 1 页的 `<title>` 与 `<h1>` 不加「第 1 页」；第 2 页以后可在 `<title>` 尾端补「－第 2 页」，避免大量分页页 title 重复。

### 场景二：无限滚动（Infinite Scroll）

无限滚动对用户体验好，但对 SEO 是陷阱——如果内容只靠 JS 滚动加载、没有对应的可抓取网址，谷歌经常只收录到第一批内容。正确做法：

1. 搭配 **History API**（历史记录接口），滚动时同步把网址更新成可分享、可收藏的分页网址（如 `?page=2`），而不是永远停在同一个网址。
2. 每个滚动区块在后端也要有对应的传统分页网址存在，供谷歌爬虫直接抓取——无限滚动是「用户体验层」，底下要有分页网址当「抓取层」。
3. 评论区、商品列表这类容易做成无限滚动的区块，若技术资源有限，宁可退回「加载更多」按钮加可抓取分页，也不要做纯滚动无网址更新的版本。

### 场景三：加载更多（Load More 按钮）

- 按钮背后应该是**链接到分页网址**（`<a href="?page=2">`），用 JS 拦截点击做局部更新，而非纯用 JS fetch 接口换数据、网址完全不变。
- 这样谷歌爬虫即使不执行点击互动，也能直接跟随 `href` 抓到后续页面。
- 建议在页面最下方（页脚上方）保留纯文字的分页链接列（1、2、3…），作为「保底」的可抓取路径，即使 JS 执行失败也不影响收录。

## 分页处理方式比较

| 场景 | canonical 指向 | 抓取路径 | 常见错误 |
|---|---|---|---|
| 纯列表分页 | 自己 | `<a href>` 分页链接 | 全部 canonical 指回第 1 页 |
| 无限滚动 | 各分页自己网址 | History API 更新网址加后端分页网址并存 | 网址永不改变，内容只存在 JS 内存 |
| 加载更多 | 各分页自己网址 | `href` 链接加 JS 拦截 | 纯 JS fetch，无 `href` 可跟随 |

## 分页与重复内容的界线

分页页面彼此内容相似（同一批商品/文章的不同区段），但**不算重复内容**，因为每页展示的项目不同——只要不做「全部 canonical 指回第 1 页」这种错误合并，谷歌能正确理解这是同一系列的不同分页，不会触发重复内容惩罚。真正该注意的是**分页页面本身不要叠加额外的重复内容**（例如每页都复制一段一模一样的分类介绍文字），这会让分页页看起来像低价值的模板页。

## 常见问题（FAQ）

**Q1：分页页面该不该加 noindex？**
不建议全面 noindex。第 2 页以后如果本身有独立、有价值的商品或文章，noindex 会让它们永远没有排名机会。只有纯粹重复、内容价值极低的深分页（例如第 20 页以后几乎没人点）才考虑 noindex，但仍要保留 `follow` 让链接权重能传递。

**Q2：rel=next/prev 已经没用了，现在要不要删掉？**
可以保留也可以删，谷歌直接忽略这组标签，不会扣分也不会加分。但如果站点仍在用旧版程序自动输出，不需要特地花工程资源去移除；重点是不要「只靠它」而忘了做自我 canonical。

**Q3：无限滚动一定不能用吗？**
可以用，但必须搭配 History API 同步更新网址，并确保后端存在对应的传统分页网址供抓取。纯视觉滚动、网址永远不变的版本，是最容易让深分页内容完全不被收录的做法。

**Q4：分页页面要不要都写进 sitemap？**
通常不需要。把第 1 页（或支柱页）放进 sitemap 即可，让谷歌爬虫透过分页链接自然发现后续页面；把几十页分页全塞进 sitemap 反而会稀释 sitemap 的信号权重。

# 国际站迁移：换网址结构时如何保住多语权重不掉

**国际站迁移（例如从子域名改子目录、或全站换网址结构）最容易在两个地方漏接：301 对映没有逐语言一一对应、hreflang 集合换了网址却没重新连接。这两者只要漏一个，多语页面就会在 2–8 周内掉排名。** 正确做法是先做逐语言网址对映表，再分批切换并每天盯 GSC，而不是一次全站上线。

## 为什么国际站迁移风险比单语网站高好几倍？

单语网站迁移只要顾好一份 301 清单。国际站则要同时处理 N 种语言 × 每种语言的 URL 结构，任何一个语言版本的 301 或 hreflang 出错，都可能拖累其他语言版本——因为 hreflang 集合是"一组互相参照"的标签，只要集合里有一个网址失效或未更新，Google 官方说明就指出整组标注可能被判定无效并回退为忽略 hreflang。实务上我们看过的案例：三语站（繁体中文／简体中文／英文）迁移时只对映了英文版 301，两个中文版本沿用旧网址结构两周，结果三个语言版本的自然流量在迁移后第 3 周合计下滑 34%，直到补上对映才在第 7 周回稳。

## 逐语言网址对映表怎么做？

不要用"规则式对映"（例如写一条正则表达式套用全站），要**逐页、逐语言列出对映表**，至少包含以下字段：

| 字段 | 说明 | 示例 |
|---|---|---|
| 旧网址 | 迁移前完整 URL | `zh.example.com/products/shoes` |
| 新网址 | 迁移后完整 URL | `example.com/zh-hans/products/shoes/` |
| 语言码 | 对应的 hreflang 值 | `zh-Hans` |
| 301 状态 | 是否已设定且测试通过 | 待测 / 已测 |
| hreflang 是否更新 | 新网址是否已进新的 hreflang 集合 | 是 / 否 |
| 内容是否一致 | 迁移过程有无被误改内容 | 是 / 否 |

对映表建议用表格软件管理，迁移前先跑一次全站爬虫（如 Screaming Frog）导出所有语言版本的既有网址清单，逐一对到新网址，缺一条都可能变成孤儿页。

## 分批切换：一次全上线 vs. 分批，差在哪？

一次全站切换最大的风险是：一旦某语言的 301 或 hreflang 有系统性错误，全站同时受影响，难以定位是哪个环节出问题。分批切换的建议顺序：

1. **第 1 批（第 1 周）**：流量最小的 1–2 个语言版本先上线，验证 301 对映、hreflang 回连、canonical 是否正确。
2. **第 2 批（第 2–3 周）**：确认第 1 批 GSC 收录与排名无异常后，切换次要语言版本（通常是次要市场语言）。
3. **第 3 批（第 4 周起）**：最后才切换流量最大的语言版本（多半是主市场语言），此时对映流程已在前两批验证过，风险最低。
4. **收尾（第 5–8 周）**：旧网址 301 保留至少 6 个月，逐周检查 GSC"页面索引"报表确认新网址逐步取代旧网址。

## 迁移期间 GSC 监控清单

- **每日**：检查"页面索引"报表的"已发现－目前未编入索引"与"重复网址，Google 选择了不同的规范网址"是否异常增加，这两项是 301/canonical 没对上的早期信号。
- **每 3 天**：用"网址检查工具"抽查 5–10 个新网址，确认 Google 读到的 canonical 与 hreflang 是你预期的值。
- **每周**：比对各语言版本的"效果"报表曝光数周环比变化，任一语言版本曝光单周跌超过 20% 即列入排查。
- **迁移后第 4、8、12 周**：分别做一次全站 hreflang 扫描（成对回连、无孤立标签），确认没有随时间又出现新的断链。

## 常见问题（FAQ）

**Q1：国际站迁移后排名多久会回稳？**
若 301 与 hreflang 对映正确，多数语言版本会在 2–6 周内回到迁移前水平；若中间发现漏接并修正，回稳时间通常会延后 2–4 周。

**Q2：可以用单一正则表达式规则做 301 对映吗？**
小型单语网站可以，但国际站不建议：不同语言版本的 URL 结构、既有历史网址往往不完全对称，规则式对映容易漏掉例外页面，逐语言列表虽然费工但可控。

**Q3：旧网址的 301 要保留多久？**
建议至少 6 个月，部分语言市场（尤其是外部链接多、收录慢的市场）建议保留 12 个月以上，确认外部链接与爬虫行为都已转向新网址再考虑下线。

**Q4：迁移时可以同时更换网址结构又改版内容吗？**
不建议。同时变动 URL 结构与内容，会让排名下滑的原因难以归因；正确做法是先完成纯网址迁移并观察 2–4 周稳定后，再进行内容改版。

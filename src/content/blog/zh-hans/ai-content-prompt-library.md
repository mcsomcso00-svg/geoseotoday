# 建立可复用的内容 prompt 库：让产线稳定输出高质量草稿

**内容 prompt 库的核心价值，是把「研究、大纲、写作、润稿」四个阶段的 prompt 各自模板化并版本编号，取代每次临场手打——这让产线输出的质量变得可预期、可追溯，也可迭代。** 没有 prompt 库的团队，每篇文章的质量波动主要来自「今天谁在写、心情好不好」，而不是内容本身难度；有 prompt 库之后，波动来源会收敛到「这个版本的模板成熟度」，这是可以被系统性改善的变量。

## 为什么「随手写」是质量波动的主因

多数团队没有把 prompt 当资产管理，而是每次开新任务就临场打字。这造成三个具体问题：

1. **同一件事，今天和上周的 prompt 长得不一样**。措辞、要求的结构、给的示例都不同，模型输出自然跟着漂移。
2. **好用的 prompt 没有留下来**。某次无意间写出效果很好的 prompt，隔周要用时已经想不起确切措辭，只能凭印象重写，效果打折。
3. **没有版本记录，出问题无法回溯**。当某一批文章的验收通过率突然下滑，团队往往不知道是「这批题目比较难」还是「上周偷偷改了 prompt」，因为改动没有留痕。

这三个问题共同的解法，就是把 prompt 当成代码一样管理：模板化、版本编号、变更留痕。

## 四阶段 prompt 模板：对应内容产线的四个工站

一条健康的内容产线可拆成研究、大纲、写作、润稿四个阶段，每阶段用途不同，prompt 也应该分开维护，不要用一个万用 prompt 贯穿全程：

| 阶段 | 目的 | 输入 | prompt 应固定的部分 |
|---|---|---|---|
| 研究 | 收集 SERP 前 10 名观点、必答问题、可用数据点 | 目标关键词、意图类型 | 输出格式（结构化清单）、资料来源要求 |
| 大纲 | 把研究结果转成 H2 结构与内链指示 | 研究输出、Boss 页清单 | 答案优先段落规则、内链位置规则 |
| 写作 | 依大纲写出完整初稿 | 大纲、品牌语气示例 | 语气示例、禁用词清单、FAQ 格式 |
| 润稿 | 事实核查、补第一手经验、删空话 | 初稿、核查来源 | 核查清单、E-E-A-T 补强提示 |

四阶段分开的好处：某一阶段效果不好时，只需要修那一阶段的模板并升版，不会牵动其他三个已经稳定的模板。这也呼应了拆分模型工作流的原则——不同工作用不同工具，prompt 同理。

## 命名与版本管理规则

没有规则的文件夹很快变成「final_v2_真的最终版.txt」的灾难。建议采用固定命名格式：

```
[阶段]_[用途]_v[版本号]_[修改日期].md
例：writing_blogpost_v3_20260812.md
```

每次修改 prompt，遵守三条规则：

1. **升版不覆盖**。v2 效果不好想改，另存 v3，不要直接改写 v2 的文件内容——旧版本要能随时调出来比对。
2. **每个版本附一行修改说明**。例如「v3：加入禁止使用『总结来说』等 AI 惯性收尾语」，让半年后的自己看得懂当初为什么改。
3. **每篇文章记录使用的 prompt 版本号**。这样当某批文章验收分数下滑，回头查是哪个版本出的问题只要几分钟，而不是重新盘点整批文章的写作过程。

## 一份可直接套用的写作阶段 prompt 骨架

以「写作」阶段为例，一份成熟的模板通常包含五个固定区块：

1. **角色设定**：你是谁、写给谁看（例如「你是 B2B SaaS 领域的 SEO 内容写手，读者是营销主管」）。
2. **输入资料**：大纲、目标关键词、必须包含的数据点清单。
3. **结构要求**：答案优先段落、H2 拆分逻辑、FAQ 格式、表格或清单至少一项。
4. **语气与禁用词**：附 2–3 段既有文章当语气示例；列出禁用的 AI 惯性用语（如「总结来说」「值得注意的是」开头句）。
5. **输出格式**：纯 Markdown、字数区间、不要加任何模板外的免责声明或客套开场。

把这五区块写死在模板里，每次任务只替换第 2 项的输入资料，其余固定不变——这正是「质量可预期」的机制来源：变动范围越小，输出的一致性越高。

## 常见问题（FAQ）

**Q1：prompt 库要用什么工具管理？**
不需要专门工具，一个有版本历史的文件夹（例如 Git 仓库或云盘的版本记录功能）就足够。重点不是工具，是有没有落实「升版不覆盖、附修改说明」这两条规则。

**Q2：四个阶段的 prompt 一定要分开吗？合并成一个不行吗？**
技术上可以，但合并后很难单独迭代。研究阶段需要的是资料收集能力，写作阶段需要的是语感，两者优化方向不同；分开维护才能各自针对弱点调整，不会因为改了写作语气而意外影响研究阶段的输出格式。

**Q3：新人加入产线，要怎么快速上手 prompt 库？**
给新人看最新版本的四份模板加上各自的修改说明历史，让他理解「为什么现在长这样」，而不是只丢一份没有脉络的最终版模板——理解演变过程能让新人更快抓到调整的边界在哪里。

**Q4：prompt 模板多久该检视一次？**
建议每批文章验收后顺手看一次通过率，若连续下滑就是模板该升版的信号；没有异常时，至少每季系统性检视一次，因为模型版本更新或市场语境变化都可能让旧模板效果衰退。
